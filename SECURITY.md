# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| main branch | ✅ Yes |
| older tags | ✗ No |

## Reporting a Vulnerability in APEX Itself

If you find a security vulnerability **in APEX's own code** (not in a target you are fuzzing with it), please **do not open a public GitHub issue**.

Instead:
1. Open a [GitHub Security Advisory](https://github.com/YOUR_USERNAME/apex-fuzzer/security/advisories/new) (private, only visible to maintainers)
2. Include a description of the vulnerability, steps to reproduce, and potential impact
3. We will respond within 7 days and coordinate a fix + public disclosure

## Responsible Use

APEX is a security research tool. You are solely responsible for ensuring you have proper authorization before fuzzing any system. Running APEX against systems without explicit written permission may violate computer crime laws including the CFAA (US), Computer Misuse Act (UK), and equivalent laws in your jurisdiction.
